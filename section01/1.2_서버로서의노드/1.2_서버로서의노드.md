# 서버로서의 노드

- 노드를 서버로 사용할 때 특성과 장단점을 알아본다.

- 노드는 기본적으로 싱글 스레드, 논블로킹 모델을 사용, 노드 서버 또한 동일한 모델일 수밖에 없다.
- 노드 서버의 장단점은 싱글 스레드, 논블로킹 모델의 장단점과 크게 다르지 않다.

- 서버는, 기본적으로 I/O 요청이 많이 발생, I/O 처리를 잘하는 노드를 서버로 사용한다.
- 노드는 (논블로킹 방식 코드 작성 가정) libuv 라이브러리를 사용해 I/O 작업을 논블로킹 방시으로 처리한다.
- 스레드 하나가 많은 수의 I/O를 혼자서 감당할 수 있다.
- 노드는 CPU 부하가 큰 작업에는 적합하지 않다.
- 모두 스레드 하나에서 처리되고 코드가 CPU연산을 많이 요구하면 스레드 하나가 혼자서 감당하기 어렵다.

- 데이터를 실시간으로 주고 받는데 적합하다.
- 네트워크, 데이터베이스, 디스크 작업 같은 I/O 에 특화되어 있다.
- 실시간 채팅 애플리케이션, 주식 차트, JSON 데이터를 제공하는 API 서버가 노드를 많이 사용한다.

- 워커 스레드 기능의 안정화로 멀티 스레드 작업을 할 수 있으나 싱글 스레드에 비해 난이도가 높다.
- 서버로는 권장하지 않는다.
- 노드로 하고 싶으면, 람다, 구글 클라우드 같은 서비스에서 노드로 사용하는 작업을 처리하는 걸 고려할 수 있다.

- 노드는 웹 서버가 내장되어 있어 쉽게 접근할 수 있다.
- 프로그래밍 외에도 웹 서버와 was 사용법을 익혀야 한다.
- 서버 규모가 커지면 nginx 등의 웹 서버를 노드 서버와 연결해야 한다.
- 노드의 가장 큰 장점은 자바스크립트를 언어로 사용한다는 것이다. 웹 브라우저도 자바스크립트를 사용하여 서버까지 노드를 사용하면 하나의 언어로 웹 사이트를 개발할 수 있다.

**노드의 장단점**

- 장점:

  1. 멀티 스레드 방식에 비해 적은 컴퓨터 자원 사용
  2. I/O 작업이 많은 서버로 적합하다.
  3. 멀티 스레드 방식보다 쉽다
  4. 웹 서버가 내장되어 있다.
  5. 자바스크립트를 사용한다
  6. JSON 형식과 쉽게 호환된다.

- 단점:

  1. 기본적으로 싱글 스레드라서 CPU 코어를 하나만 사용한다
  2. CPU 작업이 많은 서버로 부적합하다
  3. 하나뿐인 스레드가 멈추지 않도록 관리가 필요하다
  4. 서버 규모가 커졌을 때 서버를 관리하기 어렵다.
  5. 성능이 애매하다.

- 쇼핑몰, 블로그 같은 웹 사이트는 정적인 콘턴체르르 많이 제공한다.
- 기본적인 틀이 있고 안의 내용물(텍스트, 이미지)만 조금씩 달라진다.
- 노드가 다른 서버에 비해 이런 콘텐츠를 제공하는데 뚜렷한 장점은 없으나 탬플릿 엔진을 다뤄 다른 언어와 비슷하게 콘텐츠를 제공할 수 있다.
- 안전성과 보안성 측면에도 충분히 검증되었다.
